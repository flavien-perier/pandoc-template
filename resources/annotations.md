

[^PRODUCT_OWNER]: Dans la méthode de projet Scrum le product Owner est un spécialiste du domaine sur lequel va porter l'application. Il va donc servir de référent.
[^DATA_LAKE]: Un data-lakes est une architecture dans laquelle les données de plusieurs applications sont centralisées en un même espace de stockage à des fins d'analyses.

[^SGBD]: Le sigle SGBD signifie: Système de Gestion de Base de données.
[^SQL]: Le SQL (Structured Query Language) est un langage permettant d'interroger des bases de données relationnelles.
[^NOSQL]: Les bases de données NoSQL (Not Only SQL) sont des bases de données utilisant des schémas de données différents de ceux des bases de données relationnelles classiques. On trouve notamment parmi elles les bases de données graphe, clé-valeur, en colonne et document.

[^NEO4J]: Neo4J est une base de données NoSQL stockant les données sous forme de graph.
[^SPARK]: Framework BigData permettant d'effectuer des traitements sur différentes sources de données.
[^NIFI]: Framework BigData de manipulation de flux de données.
[^KAFKA]: Kafka est une file de messages asynchrones. Elle permet à plusieurs applications d'échanger des messages à travers le réseau.

[^LDAP]: L'annuaire LDAP de l'entreprise Orange contient la liste de toutes les personnes travaillant au sein de l'entreprise avec leur profil.
[^LEMON_LDAP]: LemonLdap est un SSO Open source. C'est par exemple lui qui est utilisé pour le système FranceConnect qui permet aux citoyens français de se connecter aux différentes administrations avec les mêmes identifiants. 
[^DOCKER]: Application permettant de manager des conteneurs.

[^DATA_VIZ]: La dataviz (visualisation de données) consiste à mettre en forme des données sous forme de graphique afin de faire ressortir des informations.
[^DEPLOIEMENT_CONTINUE]: Le déploiement continu consiste à faire en sorte que lors du développement d'une application des phases de tests automatiques et successives passent afin de valider le code produit par les développeurs et de le déployer en intégration, voire en production automatiquement.

[^FRONT_END]: Le Front-end d'une application est la partie chargée de gérer l'affichage.
[^BACK_END]: Le Back-end d'une application est la partie qui a pour responsabilité d'effectuer les traitements.

[^IAAS]: Les plateformes IAAS (Infrastructure as a service) offrent à leurs utilisateurs des machines virtualiser dans le cloud sur lesquelles il est possible d'installer leurs différentes applications.
[^PAAS]: Les plateformes PAAS (Plateforme as a service) ont pour but de proposer aux développeurs un système pour déployer leurs applications dans le cloud en s'affranchissant des contraintes système et réseau.
[^CAAS]: Les plateformes CAAS (Container as a service) permettent de faciliter le déploiement de conteneurs dans le cloud.
[^SAAS]: Les solutions SAAS (Software as a service) offrent la possibilité d'utiliser une application directement dans le cloud.

[^CSV]: Le CSV est un format de fichier en colonnes, où les données sont séparées par un caractère servant de délimiteur (en général le point-virgule).
[^JSON]: Le Json est un format de fichier clé valeur.
[^XML]: Le XML est un format de fichier structuré.
[^JVM]: La JVM (Java Virtual Machine) permet d'exécuter une application Java.

[^GDPR]: LA GDPR ou RGPD (Règlement général sur la protection des données) est une loi européenne traitant de la confidentialité des données des utilisateurs.
[^ETL]: Les ETL (Extract Transform Load) sont des technologies permettant de synchroniser plusieurs bases de données entre elles.
[^SSO]: Un SSO (Single Sign-On) est un proxy d'authentification unique permettant de se connecter sur plusieurs applications avec les mêmes identifiants.
[^POC]: Proof Of Concept (Preuve de concept) micro projet préliminaire ayant pour but de prouver la faisabilité d'un projet.
